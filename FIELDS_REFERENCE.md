# 字段数据源和配置参考文档

本文档详细说明了 Data Fill 插件中所有预设字段的数据源、配置项和生成逻辑。

---

## 📋 文档说明

- **数据源**：字段使用的原始数据（如姓氏列表、城市列表等）
- **配置项**：用户可以自定义的参数
- **生成逻辑**：数据如何生成的算法说明
- **代码位置**：相关代码在项目中的位置

## 🔤 文本字段

### 1. 人员姓名 (person_name)

**字段 ID**: `person_name`  
**类型**: 文本  
**标签**: 随机、中文

**数据源**:
```javascript
// code.ts 第 24-25 行
const SURNAMES = ['王', '李', '张', '刘', '陈', '杨', '黄', '赵', '周', '吴'];
const GIVEN_NAMES = ['伟', '芳', '娜', '秀英', '敏', '静', '丽', '强', '磊', '军'];
```

**生成逻辑**:
- 随机选择一个姓氏
- 随机选择一个名字
- 组合成完整姓名（如"王伟"）

**配置项**: 无

**示例输出**: `王伟`, `李娜`, `张静`

---

### 2. 日期时间 (date_time)

**字段 ID**: `date_time`  
**类型**: 文本  
**标签**: 随机、日期

**数据源**:
- 年份范围: 2020-2026
- 月份范围: 1-12
- 日期范围: 1-28

**生成逻辑**:
- 随机生成年、月、日
- 根据配置的格式输出

**配置项**: ✅ 可配置
- **日期格式** (format):
  - `YYYY-MM-DD` (默认): 如 2024-01-15
  - `YYYY/MM/DD`: 如 2024/01/15
  - `MM-DD`: 如 01-15

**示例输出**: 
- `2024-01-15` (YYYY-MM-DD)
- `2024/01/15` (YYYY/MM/DD)
- `01-15` (MM-DD)

**代码位置**: `code.ts` 第 60-71 行

## 🔧 如何修改数据源

### 修改现有数据源

**示例：添加更多姓氏**

1. 打开 `code.ts`
2. 找到第 24 行的 `SURNAMES` 数组
3. 添加新的姓氏：
```typescript
const SURNAMES = ['王', '李', '张', '刘', '陈', '杨', '黄', '赵', '周', '吴', '孙', '马'];
```
4. 编译 TypeScript: `npx tsc`
5. 重新加载插件

### 添加新的配置项

**示例：为"手机号码"添加号段配置**

1. **修改 UI 配置面板** (`ui.html`)
   - 在 `renderConfigOptions()` 函数中添加新的 case
   - 添加配置表单 HTML

2. **修改配置保存逻辑** (`ui.html`)
   - 在 `saveConfig()` 函数中添加新的 case
   - 读取配置值并保存

3. **修改数据生成逻辑** (`code.ts`)
   - 在 `generateFieldData()` 函数中使用配置
   - 应用配置到数据生成

4. **更新本文档**
   - 在对应字段下添加配置项说明
   - 更新配置项汇总表

## 📝 维护说明

### 新增字段时需要更新的位置

1. **字段定义** (`ui.html`)
   - 在 `DEFAULT_FIELDS` 中添加字段定义
   - 设置字段 ID、名称、图标、标签等

2. **数据生成器** (`code.ts`)
   - 在 `generateFieldData()` 函数中添加 case
   - 实现数据生成逻辑

3. **配置选项**（如果需要）
   - UI: `renderConfigOptions()` 和 `saveConfig()`
   - code.ts: 应用配置到生成逻辑

4. **本文档**
   - 添加字段详细说明
   - 更新配置项汇总表

### 修改数据源时需要注意

- 保持数据的多样性和真实性
- 考虑数据的文化适配性（如中文姓名）
- 确保数据格式的一致性
- 测试数据生成的随机性
