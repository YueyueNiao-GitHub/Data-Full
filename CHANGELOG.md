### 🔧 修复：序号自增和错误提示问题（已完成）

**问题 1**：批量填充序号时，所有图层显示相同数字  
**问题 2**：图层类型不匹配时，没有 UI 错误提示

**原因分析**：
- 问题 1：UI 生成一次数据后发送给所有图层，导致序号相同
- 问题 2：错误信息只在控制台显示，用户看不到

**解决方案**：
- ✅ 将数据生成逻辑从 UI 移到 `code.ts`
- ✅ 每个图层独立调用数据生成函数
- ✅ 序号计数器在 `code.ts` 中维护，确保自增
- ✅ 添加 UI 错误提示弹窗（红色 Toast，2秒自动消失）
- ✅ 添加 `padStart` polyfill 以兼容 TypeScript

**文件变更**：
- `code.ts`：添加完整的数据生成器代码，每个图层独立生成数据
- `ui.html`：移除数据生成调用，只发送字段 ID；添加错误提示 UI

**状态**：🟢 已完成

**验收方式**：
1. 创建 3 个文本图层，全选，点击"排列序号" → 显示 1、2、3
2. 创建矩形图层，选中，点击文本字段 → 显示红色错误提示"格式不匹配：只能填充文本图层"

### 🔧 修复：配置存储问题（已完成）

**问题**：配置面板可以打开，但无法保存配置

**原因**：Figma 插件的 iframe 环境不支持 `localStorage`，导致 `SecurityError`

**解决方案**：
- ✅ 改用 Figma 的 `clientStorage` API 存储配置
- ✅ UI 通过消息传递将配置发送给 code.ts
- ✅ code.ts 使用 `figma.clientStorage.setAsync()` 保存配置
- ✅ 插件启动时自动加载配置并发送给 UI
- ✅ UI 使用内存缓存存储配置

**文件变更**：
- `ui.html`：移除 localStorage，改用消息传递和内存缓存
- `code.ts`：添加 `loadConfigs()` 和 `saveConfig()` 函数

**状态**：🟢 已完成
